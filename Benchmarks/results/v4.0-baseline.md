# v4.0 baseline — `SerializerComparisonBench` (smoke run, `--job short`)

Captured by Agent F as part of Phase 6 (Chunk 6) of the v4.0 binary serialization migration.
Run is `--job short` (IterationCount=3, LaunchCount=1, WarmupCount=3) — values are smoke-quality,
not publication-quality. The point is shape and order-of-magnitude.

## Environment

- BenchmarkDotNet v0.14.0
- Windows 11 (10.0.26200.7623)
- .NET SDK 10.0.203, .NET 10.0.7 (X64 RyuJIT AVX2)
- GC = Concurrent Server

## Observations (smoke-quality only)

- XPacket Envelope_Serialize is ~2.5x faster than Json and allocates ~2.5x less.
- XPacket Fragment_OrderDto is ~4x faster than Json and allocates ~2.4x less.
- Envelope_Deserialize gap is ~3.7x in favor of XPacket.

A full publication-quality run would drop `--job short` and pin the CPU. This file
is reference-grade for the v4.0 migration commit and is intentionally tracked despite
`Benchmarks/results/` being gitignored by default.
